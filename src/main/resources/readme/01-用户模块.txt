一、用户登录
    后端实现
        整体思路：
            1. 参数判断
                用户姓名、用户密码   非空判断
            2. 通过用户名查询用户记录，返回对象
            3. 判断返回的用户对象是否为空
            4. 如果用户对象不存在，抛出异常
            5. 判断密码是否正确，正确的话登录成功

        Controller 层(控制层，接受请求，响应结果)：
            1. 通过形参的方式接受数据
            2. 调用业务逻辑层的登录方法，得到登录结果
            3. 响应数据给客户端

        Service 层(业务逻辑层，非空判断，条件判断等逻辑判断)
            1. 参数判断，不正确抛出异常
            2. 调用数据访问层，返回查询结果
            3. 结果判断，异常则抛出
            4. 对比对象的密码和前端传来的密码
            5. 返回结果

        dao 层(数据访问层，操作数据库)
            根据 service 传递过来的参数操作数据库，返回去结果
